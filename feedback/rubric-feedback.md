# Rubric 排序权重 · 反馈调参日志

> 这是排序权重（`config.yaml` → `ranking.weights`：`semantic` / `popularity`）的**人工反馈调参记录**。
> live 权重的唯一来源仍是 `config.yaml`；本文件只记录**为什么这么调**，按时间倒序累积。
> 后续可能升级成专门的 skill（用户反馈 → 建议调整 → 自动改 config），目前先用这个本地文件。

## 怎么用

每次用户针对排序效果提反馈时，**在「调参记录」顶部新增一条**，五要素填全：

| 字段 | 含义 |
|---|---|
| 日期 | YYYY-MM-DD |
| 场景 | 在什么 query / 什么结果下触发的反馈 |
| 用户反馈 | 用户的原话或诉求 |
| 系统建议 | 据反馈推断该往哪个方向调（语义↑还是热度↑），及理由 |
| 实际修改 | 对 `config.yaml` 的改动 `旧值 → 新值`；若只记录不改则写「未调整」 |

权重调整经验法则：
- 抱怨「找来的人跟需求不对路 / 不相关」→ 提 `semantic`（语义）。
- 抱怨「找来的都是小号 / 不够头部 / 没流量」→ 提 `popularity`（热度）。
- 两者和保持为 1.0。
